# 🚀 个人成长仪表板部署指南

## 🌟 项目特色功能

### 🎊 年度总结功能
- **智能关键词分析**: 自动提取全年记录中的高频词汇
- **全面数据统计**: 汇总全年的记录天数、运动时长、学习时长等
- **成长轨迹回顾**: 按月展示成长里程碑
- **成就系统**: 6种不同类型的成就徽章
- **个性化寄语**: 基于用户数据生成专属年度总结
- **12月31日自动提醒**: 年末自动提醒生成年度总结
- **一键导出**: 支持将年度总结导出为高清图片

## 📋 部署方式选择

## 🌐 GitHub Pages 部署

### 1. 创建GitHub账号
如果还没有GitHub账号，请先到 [github.com](https://github.com) 注册

### 2. 创建新仓库
1. 登录GitHub后，点击右上角的 "+" 号
2. 选择 "New repository"
3. 仓库名称建议：`growth-dashboard` 或 `personal-growth-tracker`
4. 设置为 **Public**（公开仓库才能使用免费的GitHub Pages）
5. 勾选 "Add a README file"
6. 点击 "Create repository"

### 3. 上传文件
有两种方式上传文件：

#### 方式A：网页上传（推荐新手）
1. 在仓库页面点击 "uploading an existing file"
2. 将以下文件拖拽到上传区域：
   - `dashboard.html`（主应用文件，包含年度总结功能）
   - `index.html`（首页和二维码生成器）
   - `README.md`（项目说明文档）
   - `deploy-guide.md`（部署指南）
3. 在底部填写提交信息：`Add growth dashboard with annual summary feature`
4. 点击 "Commit changes"

#### 方式B：Git命令行
```bash
# 克隆仓库到本地
git clone https://github.com/你的用户名/growth-dashboard.git

# 进入仓库目录
cd growth-dashboard

# 复制文件到仓库目录
# 将 dashboard.html, index.html, README.md, deploy-guide.md 复制到这里

# 添加文件到Git
git add .

# 提交更改
git commit -m "Add growth dashboard with annual summary feature"

# 推送到GitHub
git push origin main
```

## 💻 本地部署

### 快速启动
```bash
# 方式1：使用Python（推荐）
python -m http.server 8000

# 方式2：使用Node.js
npx serve .

# 方式3：使用PHP
php -S localhost:8000
```

### 访问地址
- 打开浏览器访问：`http://localhost:8000`
- 年度总结功能完全可用，数据存储在浏览器本地

### 优势
- ✅ 数据完全私密，不会上传到任何服务器
- ✅ 访问速度快，无网络延迟
- ✅ 支持离线使用
- ✅ 年度总结导出功能正常工作

## ☁️ 云服务器部署

### 适用场景
- 需要自定义域名
- 多设备数据同步
- 团队或家庭共享使用

### 推荐服务商
- **阿里云**: 国内访问速度快
- **腾讯云**: 提供免费额度
- **Vercel**: 免费静态网站托管
- **Netlify**: 一键部署，自动HTTPS

### Vercel部署步骤
1. 访问 [vercel.com](https://vercel.com)
2. 使用GitHub账号登录
3. 点击 "New Project"
4. 选择你的GitHub仓库
5. 点击 "Deploy"
6. 几分钟后获得访问链接

---

### 4. 启用GitHub Pages
1. 在仓库页面点击 "Settings" 标签
2. 在左侧菜单找到 "Pages"
3. 在 "Source" 部分选择 "Deploy from a branch"
4. 选择 "main" 分支
5. 文件夹选择 "/ (root)"
6. 点击 "Save"

### 5. 获取访问链接
- 几分钟后，你的网站将在以下地址可用：
- `https://你的用户名.github.io/仓库名称`
- 例如：`https://zhangsan.github.io/growth-dashboard`

## 📱 二维码生成

网站部署成功后，你可以：

1. **自动生成**：访问 `index.html` 页面会自动显示二维码
2. **手动生成**：使用在线二维码生成器
   - 推荐：[qr-code-generator.com](https://www.qr-code-generator.com/)
   - 输入你的GitHub Pages链接
   - 下载二维码图片

## 🔧 自定义域名（可选）

如果你有自己的域名：

1. 在仓库根目录创建 `CNAME` 文件
2. 文件内容写入你的域名，如：`growth.yourdomain.com`
3. 在域名DNS设置中添加CNAME记录指向 `你的用户名.github.io`

## 📊 使用统计

GitHub Pages提供基本的访问统计：
- 在仓库页面点击 "Insights" → "Traffic"
- 可以看到访问量和访客数据

当你需要更新工具时：

2. 重新上传到GitHub仓库
3. GitHub Pages会自动更新（通常需要几分钟）

## 🎯 年度总结功能使用指南

### 📊 功能亮点
- **智能分析**: 自动分析全年数据，生成个性化报告
- **关键词云**: 基于记录内容提取高频词汇
- **成就系统**: 6种成就类型，根据实际数据解锁
- **可视化展示**: 美观的图表和时间轴展示
- **一键导出**: 支持导出为高清图片分享

### 🔧 使用步骤
1. **访问功能**: 在数据面板页面点击"年度总结"按钮
2. **选择年份**: 使用下拉菜单选择要分析的年份
3. **生成报告**: 点击"生成年度总结"，系统自动分析数据
4. **查看内容**: 浏览关键词、统计数据、成就、成长轨迹
5. **导出分享**: 点击"导出图片"保存为PNG格式

### 🏆 成就系统说明
- **坚持记录者**: 全年记录天数≥200天
- **运动达人**: 全年运动时长≥100小时
- **学习之星**: 全年学习时长≥200小时
- **阅读爱好者**: 全年阅读书籍≥12本
- **情绪管理大师**: 平均情绪分数≥4分
- **全面发展**: 五个模块都有记录

### 🎨 个性化特色
- **智能寄语**: 根据用户数据生成专属年度总结语
- **成长轨迹**: 按月展示重要里程碑
- **数据洞察**: 发现个人成长模式和趋势

## ❓ 常见问题

**Q: 为什么网站打不开？**
A: 请检查：
- GitHub Pages是否已启用
- 文件是否正确上传
- 等待几分钟让部署完成

**Q: 可以使用私有仓库吗？**
A: GitHub Pages对私有仓库需要付费订阅

**Q: 如何统计访问量？**
A: 可以集成Google Analytics或使用GitHub自带的统计功能

**Q: 网站加载慢怎么办？**
A: GitHub Pages在国内访问可能较慢，可以考虑使用CDN加速
